# PROJECT_BRIEF.md

## Product Name

VibeVerse

## One-Line Pitch

VibeVerse is an AI-native interactive music discovery platform where users explore their music taste as a living galaxy, generate emotional playlists through an AI DJ, save memories to songs, and join realtime collaborative vibe rooms.

## Core Experience

The app should not feel like a basic Spotify clone.

It should feel like:
- A music universe
- A personal taste map
- An AI DJ
- A social listening room
- A music memory journal

## Main Features

### 1. AI Playlist Generator

Users can type prompts like:

> Make me a playlist that feels like late-night coding in Chicago during winter.

The app should generate a playlist concept, explain the vibe, and suggest tracks/artists using available metadata.

### 2. Music Memory Journal

Users can attach personal memories to songs.

Examples:
- This song reminds me of college.
- This was my gym PR song.
- This reminds me of Delhi night drives.

The AI should later retrieve and summarize these memories.

### 3. Music Galaxy

A visual interface where:
- Artists are planets
- Genres are constellations
- Songs are particles
- Users can explore relationships visually

The first version can be 2D. Later versions can use 3D/WebGL.

### 4. Taste Profile

The app should analyze saved tracks, memories, genres, moods, and artists to produce a Taste DNA profile.

### 5. Vibe Rooms

Realtime rooms where users can:
- Join a shared room
- Add songs to a queue
- Vote on songs
- React with moods/emojis
- Let the AI DJ explain or adjust the room vibe

## MVP Scope

The first implementation should focus on:

- Auth
- Database schema
- Track/artist search
- Saved tracks
- Music memories
- AI playlist generation
- Basic taste profile
- Basic visual music map

Do not build realtime rooms or advanced 3D visualization until the MVP is working.

## Design Direction

The app should feel futuristic, cinematic, musical, and interactive.

Keywords:
- Dark mode
- Neon gradients
- Glassmorphism
- Animated particles
- Smooth transitions
- Music visualizer feel
- Premium startup product
- Not generic SaaS dashboard

## Technical Goal

This project should demonstrate:

- Full-stack engineering
- Modern frontend architecture
- Database design
- LLM integration
- Vector search
- API integration
- Realtime systems eventually
- Strong UI/UX taste
